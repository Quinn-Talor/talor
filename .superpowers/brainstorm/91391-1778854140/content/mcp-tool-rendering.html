3未知工具 · 泛化 fallback 渲染

用户加了一个 MCP server（比如 Notion / 自建），其工具名 registry 里没有。不能不渲染，也不该乱猜。统一走 GenericMcpRenderer：tool row + 简洁 JSON 输出（默认折叠，展开看完整结果）。

设计原则：未知工具不假装专业渲染（避免 LLM 凭空补字段）；JSON 是最诚实的"我不认识但我能给你看原始数据"。常用工具一旦稳定，再升级到 registry 加专门渲染器。

核心原则

  • 能力是用户配置的 — Talor 不预装邮件 / 日历，用户按需加 MCP server
  • 渲染按 tool name lookup — 不靠 schema、不靠 LLM 判断，直接 renderers[toolName]
  • 常用 MCP 工具有专门渲染器 — Gmail / GCal / Brave / Outlook 等主流工具的 tool name 收录到 registry
  • 未知工具走 generic fallback — JSON pretty-print，诚实展示
  • Agent profile 决定可用工具集 — "秘书" agent 只能用 email + calendar；"研究助理" 只能用 web search + read_url。已在 Schema 2.0 设计中
