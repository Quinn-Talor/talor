最终决策表
Block 类型 渲染策略 底层数据
need_input 折回 prose（markdown list 即选项） 仍记 DB / IPC
blocked 折回 prose 仍记，给 retry 路由用
warning 折回 prose；高危事件去安全日志 仍记 telemetry
done 可选小尾标（长任务） 用作 turn 结束 marker
draft_detected 保留 UI（系统级 CTA） 必须，含 profile payload
invalid 不渲染 → 开发者日志 parser debug only
inferred_intent 不渲染 → 删除（产品价值低） 建议直接砍

核心信念：聊天 UI 只渲染 agent 的话 和 系统的动作邀请（draft）。其他 block 是基础设施 — 它们在 DB / IPC / metric 里活着，但不抢用户视野。
