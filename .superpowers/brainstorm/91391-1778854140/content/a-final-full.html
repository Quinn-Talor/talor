T

读一下文件再判断。

read src/main/loop/react-loop.ts 8ms
grep "seenIds" — 3 matches in 1 file 14ms

主循环用两层保险防止死循环：

  • 硬上限 — maxIterations 默认 20，越界直接 throw
  • step_id 漂移检测 — 每个 step 的 id 必须递增，重复立刻 halt

关键代码在 react-loop.ts:142：

typescript · react-loop.ts:140-146 copy 
for (let iter = 0; iter < ctx.maxIterations; iter++) {
  const step = await ctx.provider.generate(messages)
  if (seenIds.has(step.id)) {
    throw new ContextHaltError(step.id)  // 漂移即 halt
  }
  seenIds.add(step.id)
}

结论：正常情况下不会死循环。两层保险都靠代码强制，不依赖 prompt 软引导（符合 standards §J-SHOULD-1）。
