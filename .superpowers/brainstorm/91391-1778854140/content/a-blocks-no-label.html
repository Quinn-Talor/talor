本轮 diff 总结
  • done — 去掉 "Done" 字，保留绿点 + 摘要
  • warning — 去掉 "warning" 字，保留圆点（颜色定档严重度）+ body
  • blocked — 去掉 "blocked" 字，保留橙点 + body + 行内 retry 链接
  • need_input — 去掉 "need input" 字，保留 2px 蓝左竖线 + 问题 + 选项 + reason
  • draft_detected — 去掉 "draft detected" 字，保留 2px 紫左竖线 + 内容 + CTA
  • invalid / inferred_intent — 整条删除（含 hint 内容），转到开发者 log
核心原则：色点 + 容器形态 + 内容语气 = 用户能理解的 100% 信息。block type 是开发者命名，用户不需要看。
