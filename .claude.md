## §3 会话协议

### 3.1 会话启动仪式（6步）
每次开始编码会话前，必须执行：
1. **环境就绪检查**：确认工作目录、Node.js 版本、依赖服务
2. **文档版本同步**：检查依赖文档状态（OVERVIEW ≥ active, REQUIREMENTS ≥ review）
3. **实施边界确认**：明确本次会话的 Phase 范围，不超范围实现
4. **命名一致性检查**：使用 requirements.md §1.3 术语表的代码命名
5. **验证环境准备**：准备 Layer 1/Layer 2 验证工具
6. **会话目标设定**：明确本次要完成的 IMPL 和 AC

> 详细步骤见对应 Phase 的 `session-start.md`

### 3.2 会话结束仪式（4步）
每次结束编码会话时，必须执行：
1. **IMPL 状态更新**：在对应 `impl.md` 中更新任务状态
2. **AC 验证记录**：记录已完成的 Layer 1 验证证据
3. **阻塞项报告**：如有未解决问题，记录并通知用户
4. **DEFERRED 项确认**：检查 `deferred.md`，通知用户确认 pending 项

### 3.3 实施进度追踪
- **总体进度**：查看 `implementation.md §4.0` 仪表盘
- **Phase 进度**：查看各 Phase 的 `impl.md §P.1` 任务清单
- **AC 验证状态**：查看各 Phase 的 `impl.md §P.3` 验证映射

## §5 禁止事项速查

| 禁止 | 原因 | 正确做法 |
|------|------|---------|
| 修改 venv/ 目录 | Python 虚拟环境，不纳入版本控制 | 不动此目录 |
| 删除/重命名 SQLite 表字段 | 需单独迁移脚本 | 用 ALTER TABLE 新增 |
| 直接修改 employees/ 示例文件 | 作为参考模板保留 | 复制后修改 |
| talor-desktop contextIsolation=false | 安全风险 | 必须 true |
| talor-desktop nodeIntegration=true | 安全风险 | 必须 false |
| API Key 明文存储 | 凭证泄露风险 | 用 safeStorage 加密 |
| Critical Path 函数返回空数据 | 虚假通过验证 | 实现真实逻辑 |
| async 函数缺少错误处理 | 错误静默失败 | 必须有 try-catch |
| 使用 `as any` 无注释 | 类型安全破坏 | 必须附带原因注释 |
| 超范围实现功能 | 范围蔓延 | 先加入 deferred.md，等用户确认 |

## §6 Gotchas 速查（模型管理功能特定）

| ⚠️ 陷阱描述 | 正确做法 | 关联文档 |
|------------|---------|---------|
| Provider API 可能不支持模型列表查询 | 降级方案：提示用户手动输入，提供常见模型预设 | implementation.md §4.3 |
| 能力检测可能触发 API 速率限制 | 实现分批检测，添加延迟，缓存检测结果 | implementation.md §4.3 |
| 模型缓存导致信息过时 | TTL 缓存（5分钟）+ 强制刷新按钮 + 过期标记 | implementation.md §4.3 |
| 模型切换导致会话历史不兼容 | 切换时提示"开始新对话"，清空历史或创建新会话 | implementation.md §4.3 |
| 不同 Provider 的模型 ID 格式不一致 | 统一格式化：`provider_id/model_name`，存储原始 ID | implementation.md §4.3 |
| 能力检测结果可能随时间变化 | 记录检测时间戳，定期重新检测，用户可手动刷新 | implementation.md §4.3 |
| 页面布局破坏现有设计一致性 | 严格遵循现有布局模式，复用Header、EmptyState等现有组件 | requirements.md §1.5.1 |
| 模型卡片信息过载 | 核心信息优先，详情可展开，使用图标+文字组合 | feature.md §F.8 ModelCard设计 |
| 模型选择器交互复杂 | 默认简单下拉，高级功能抽屉式展开，支持键盘导航 | feature.md §F.8 ModelSelector设计 |
| 响应式设计缺失 | 为每个组件定义桌面/平板/手机三种布局 | feature.md §F.8 响应式设计 |
| 无障碍访问不支持 | 实现ARIA标签、键盘导航、屏幕阅读器支持 | feature.md §F.8 无障碍设计 |
| 加载状态不明确 | 区分短/长/超时加载，提供进度反馈和取消选项 | requirements.md §1.5.1 加载状态 |
| 颜色对比度不足 | 使用WCAG标准检查，确保文本可读性 | feature.md §F.8 颜色对比度 |

## §7 术语速查（模型管理功能）

| 术语 | 代码命名 | 定义 | 使用场景 |
|------|---------|------|---------|
| 模型列表 | `modelList` / `ModelList` | Provider 支持的所有可用模型集合 | 变量、类型定义 |
| 模型信息 | `modelInfo` / `ModelInfo` | 单个模型的详细信息 | 接口、函数参数 |
| 模型能力 | `modelCapability` / `ModelCapability` | 模型支持的功能特性 | 能力检测结果 |
| 模型选择器 | `modelSelector` | UI 组件，允许用户选择模型 | 组件名称 |
| 会话模型 | `sessionModel` | 会话当前使用的具体模型 | 会话属性 |
| 能力检测 | `capabilityDetection` | 检测模型支持哪些具体能力 | 服务函数 |
| 模型缓存 | `modelCache` | 本地缓存的模型列表信息 | 缓存管理 |

> 完整术语表见 `vibe/features/talor-desktop-model-management/requirements.md §1.3`
