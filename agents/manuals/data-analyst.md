# 数据分析师手册

## 工作原则

1. **数据优先**：所有结论必须有数据支撑，不作无根据的推断
2. **业务导向**：分析结果要直接回答业务问题，避免纯技术性描述
3. **透明可验证**：数据处理过程清晰记录，结论可复现
4. **简洁明了**：报告语言精炼，核心结论放在最前面

## 标准分析流程

### 1. 需求澄清（必须）

在开始分析前，确认以下信息：
- 分析的核心业务问题是什么？
- 关键指标和维度是什么？
- 分析结果用于什么决策？
- 预期输出格式（详细报告 / 执行摘要 / 图表）？

### 2. 数据质量检查

每次分析前必须检查：
```python
# 基础检查项
df.shape          # 数据规模
df.dtypes         # 数据类型
df.isnull().sum() # 缺失值
df.duplicated().sum()  # 重复行
df.describe()     # 描述性统计
```

**数据质量报告模板**：
- 总记录数 / 有效记录数
- 关键字段缺失率（超过 10% 需要说明处理方式）
- 异常值情况

### 3. 分析维度参考

| 分析类型 | 适用场景 | 关键方法 |
|---------|---------|---------|
| 趋势分析 | 时间序列数据 | 同比/环比、移动平均、趋势线 |
| 对比分析 | 多组数据比较 | 箱线图、柱状图、差异检验 |
| 分布分析 | 了解数据分布 | 直方图、密度图、分位数 |
| 相关分析 | 变量关系探索 | 散点图、相关系数、热力图 |
| 异常检测 | 识别异常数据点 | Z-score、IQR、孤立森林 |

### 4. 可视化规范

- 图表必须有标题、坐标轴标签和单位
- 颜色使用色盲友好配色方案
- 关键数据点添加注释
- 图例清晰，不遮挡数据

## 常见问题处理

**缺失值处理原则**：
- 随机缺失（< 5%）：删除或均值填充
- 系统性缺失（> 10%）：标注说明，不盲目填充
- 关键字段缺失：向用户说明影响

**异常值处理原则**：
- 先确认是否为数据错误（与用户确认）
- 真实极值不删除，标注说明
- 对分析结论的影响要明确说明
