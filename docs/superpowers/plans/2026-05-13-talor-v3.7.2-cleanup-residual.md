## 1. 清理 A:删 delegate_agent A2/B2

按 J-SHOULD-2 协作矩阵的反模式表第 1 行(系统用 regex 判 LLM 意图)。

### A2 — `checkInstructionCompatibility`(已删)

- 系统用 regex 抽 instruction 实体 + agent profile 文本实体,做子串匹配
- 不匹配 → `INSTRUCTION_OUT_OF_SCOPE` envelope 拒绝委托
- 假阳代价:同义词 / 别名 / 模糊概念被拒绝,父 agent 退缩
- **真正应该做这事的是 LLM**:父 agent 通过 A1 capability listing 自行判断是否委托

### B2 — `checkOffTarget`(已删)

- 系统抽 instruction 实体 + 子 agent 输出实体,子串匹配
- 不命中 → `SUBAGENT_OFF_TARGET` envelope
- 假阳代价:"查百度股价" → "BIDU ¥85" → 误判 off-target → 父 agent 重做
- **真正应该做这事的是父 agent**:看到子输出后自然语言判断是否完成 instruction

### 附带

- `entity-extractor.ts: extractEntitySet` 删除(只 A2/B2 用)
- `delegate-agent.ts: isSpecificEntity` / `collectProfileEntityText` / `CompatibilityResult` 都删
- error code `INSTRUCTION_OUT_OF_SCOPE` / `SUBAGENT_OFF_TARGET` 从 HINTS 表 + react-loop isSubagentFailureOutput 删
- delegate-agent.test.ts 中 A2 / B2 整 describe 块删

`extractEntities` / `redactEntities` 保留 — `draft-extractor.ts` 仍依赖。

## 2. 清理 B:RiskGate 路径统一

### 旧实现(两套)

```
RiskGate.gate():
├── HIGH static → return {action:'pass-to-legacy', via:'legacy'}
│   → buildTools 看到 pass-to-legacy → 走嵌入式 100 行 confirm 逻辑
│   (含 buildInputSummary / diagnoseInputMismatch / confirmTool 调用 / abort 处理)
└── 其他 → pendingBlock / fallback / memory / auto-low (Gate 内 confirm)
```

两条路径下 confirm UI 字段、abort 处理、记账方式都不一致 — 维护成本高。

### 新实现(单路)

```
RiskGate.gate():
├── HIGH static → 系统生成 summary → Gate 内 confirm (新 via='high-static')
│   ledger 内部 record (与其他路径对称)
├── LLM emit pending_confirm → Gate 内 confirm (via='pendingBlock')
├── memory 命中 → 自动通过 + ledger (via='memory')
├── 代码 regex 兜底 → Gate 内 confirm (via='fallback')
└── 无信号 → 直通 (via='auto-low')
```

buildTools 整段简化:从 ~155 行 execute 函数减到 ~50 行,只看 `action='pass'/'deny'`。

### 类型变化(破坏性)

```ts
// 之前
interface GateDecision {
  action: 'pass' | 'deny' | 'pass-to-legacy'
  via: 'pendingBlock' | 'fallback' | 'memory' | 'auto-low' | 'legacy'
}

// v3.7.2 后
interface GateDecision {
  action: 'pass' | 'deny'
  via: 'pendingBlock' | 'fallback' | 'memory' | 'auto-low' | 'high-static'
}

// SideEffectEntry.confirmed_by 类型同步
type ConfirmedBy = 'pendingBlock' | 'fallback' | 'memory' | 'auto-low' | 'high-static'
```

### Ledger schema 变化(破坏性)

```sql
-- 之前
CHECK(confirmed_by IN ('pendingBlock','fallback','memory','auto-low'))

-- v3.7.2 后
CHECK(confirmed_by IN ('pendingBlock','fallback','memory','auto-low','high-static'))
```

SQLite 不支持 ALTER CHECK,所以 `recreateSideEffectLogIfOutdated` 启动期检测 schema-drift 后 DROP + 重建。**开发期 ledger 历史数据丢弃**(项目策略 + 用户指示)。

## 7. 链路对照(v3.7.2 后)

```
工具调用
  ↓
RiskGate.gate(tool, input, ctx, confirmTool)
  ├── HIGH static (bash/write/edit)
  │     ├── 缺必填字段 → deny + diagnose summary (不弹 confirm)
  │     ├── confirm approved → pass + ledger(high-static, approved)
  │     └── confirm denied   → deny + ledger(high-static, denied)
  ├── pending_confirm block
  │     ├── memory 已批 → pass + ledger(memory, auto) [auto via]
  │     ├── confirm approved (+ remember) → pass + memory.approve + ledger(pendingBlock, approved)
  │     ├── confirm approved (- remember) → pass + ledger(pendingBlock, approved)
  │     └── confirm denied → deny + ledger(pendingBlock, denied)
  ├── fallback regex
  │     ├── confirm approved → pass + ledger(fallback, approved)
  │     └── confirm denied → deny + ledger(fallback, denied)
  └── 无信号 → pass + via='auto-low' (不记账)
  ↓
buildTools 看 action:
  ├── 'deny'  → 返 USER_DENIED envelope
  └── 'pass'  → 调 agent.toolRegistry.execute() → 工具实际执行
```

整个 confirm/记账逻辑都在 Gate 内,buildTools 只调度 — **职责清晰**。
