# Talor 架构设计

## 概述

Talor 采用基于事件的微服务架构，核心设计原则包括：
- **事件驱动**: 模块间通过类型化事件通信
- **依赖注入**: 模块配置通过依赖注入管理
- **职责分离**: 每个模块单一职责
- **接口隔离**: 清晰的模块接口定义

## 模块详细说明

### 1. 事件总线 (Bus)
- **职责**: 模块间事件通信
- **特性**: 
  - 类型化事件定义
  - 异步事件发布/订阅
  - 全局总线与会话总线隔离
- **关键类**: `Bus`, `GlobalBus`, `BusEvent`

### 2. 工具系统 (Tool)
- **职责**: 工具定义、注册和执行
- **特性**:
  - Pydantic 验证
  - 权限管理
  - 批量操作支持
- **关键类**: `Tool`, `ToolRegistry`, `ToolContext`

### 3. 会话管理 (Session)
- **职责**: 对话会话生命周期管理
- **特性**:
  - 会话隔离
  - 消息持久化
  - 状态管理
- **关键类**: `Session`, `Message`, `SessionBusManager`

### 4. 代理系统 (Agent)
- **职责**: ReAct 执行引擎
- **特性**:
  - 推理-行动循环
  - 多模型支持
  - 流式输出
- **关键类**: `AgentExecutor`, `Agent`, `Permission`

### 5. 配置系统 (Config)
- **职责**: 分层配置管理
- **特性**:
  - 环境变量支持
  - 文件配置
  - Keyring 安全存储
- **关键类**: `Config`, `KeyringManager`

### 6. 提供商系统 (Provider)
- **职责**: LLM 提供商抽象
- **特性**:
  - LiteLLM 集成
  - 多提供商支持
  - 配置热加载
- **关键类**: `Provider`, `ProviderService`

### 7. MCP 客户端 (MCP Client)
- **职责**: Model Context Protocol 集成
- **特性**:
  - FastMCP 集成
  - 工具桥接
  - 协议适配
- **关键类**: `MCP`, `MCPTools`

## 数据流示例

### 用户消息处理流程

```
1. 用户输入 → API/CLI
2. API/CLI → SessionManager.create_message()
3. SessionManager → Bus.publish(MessageCreated)
4. AgentExecutor 订阅事件 → 执行 ReAct 循环
5. ReAct 循环: Reasoning → Tool Execution → Observation
6. 工具执行结果 → Bus.publish(ToolExecuted)
7. LLM 调用 → ProviderService
8. 生成回复 → SessionManager.add_message()
9. 最终回复 → 返回给用户
```

## 扩展机制

### 插件系统
- **位置**: `src/plugin/`
- **扩展点**: 工具、提供商、事件处理器
- **加载机制**: 自动发现 + 手动注册

### 技能系统
- **位置**: `src/skill/`
- **特性**: 智能任务识别和路由
- **匹配机制**: 模式匹配 + 优先级
