## 2. 编码过程中的行为规则（10 条铁律）

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | **范围锁定**：发现"顺便加 X"→ 立即写入 DEFERRED.md，不实现 | 范围蔓延 |
| 2 | **命名单一性**：变量/函数/注释必须与 REQUIREMENTS.md §1.3 术语表一致 | 术语混乱 |
| 3 | **单步推进**：禁止同时修改多个未连接的模块 | 难以验证 |
| 4 | **连线优先**：新模块创建后必须立即连接到调用链，再建下一个 | 产生孤岛模块 |
| 5 | **错误处理完整性**：每个 async 外部调用（HTTP/FS）必须有 try-catch | 静默失败 |
| 6 | **类型安全契约**：禁止 `as any`（除非有注释说明必要原因） | 绕过类型检查 |
| 7 | **回归防护**：每次修改后重新验证已工作的 Demo 场景 | 回归 bug |
| 8 | **实施前声明**：开始任何模块前确认：输入/输出/被谁调用/用什么验证 | 方向偏差 |
| 9 | **渐进式集成**：整个实施过程中应用必须"始终可启动"（npm run dev） | 无法 Demo |
| 10 | **TDD 优先**：每个 IMPL 先写失败测试，再写最少代码通过，再重构 | 无测试覆盖 |

## 5. 禁止事项速查

| 禁止 | 原因 | 正确做法 |
|------|------|---------|
| contextIsolation=false | 暴露 Node.js API 到 renderer，安全风险 | preload 通过 contextBridge 暴露 TalorAPI |
| nodeIntegration=true | 同上 | 始终 false |
| API Key 明文写入 config.json | 凭证泄露风险 | 使用 safeStorage 加密到 api-keys.enc |
| config.json 直接 writeFile（无 rename） | 崩溃时文件损坏 | 写 .tmp 后 rename（atomic） |
| ollama 测试端点用 /v1 路径 | ollama 不用 /v1 前缀 | 用 /api/tags |
| setDefault 跳过幂等处理 | 并发时默认标签可能多个 | 先全置 false，再目标置 true，原子写入 |
| Critical Path 上返回 null / [] / 硬编码 | Demo 无法跑通 | 始终返回真实计算结果 |

## 6. 已知陷阱速查（Gotchas）

| ⚠️ 陷阱 | 正确做法 |
|---------|---------|
| Electron main process 用 CommonJS（`__dirname` 可用），renderer 用 ESM（Vite） | main: `path.join(__dirname, ...)`；renderer: `import.meta.url` |
| safeStorage 在 macOS/Linux 可用，Windows 需检测 `isEncryptionAvailable()` | 启动时检测，不可用时弹出警告 |
| config.json 写入必须 atomic rename | 写 `config.json.tmp` 后 rename |
| ollama base_url 不含 `/v1`（原生 API 用 `/api/chat`） | provider-tester 按 type 构造不同路径 |
| AbortController 取消重复测试请求 | provider-tester 维护一个 AbortController |
